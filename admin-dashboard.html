Understanding the backend architectural dependency chains and data flows for client interactions.

Cart Addition Flow

1

Client Request

App sends POST /api/cart/add containing productId, quantity, and Bearer token.

2

Auth & Validation Middleware

Verifies JWT token integrity, extracts User ID, and validates payload schema using Joi to prevent bad DB queries.

3

Product Verification

Queries Product.findOne() to ensure the item actually exists before allowing it to be queued for purchase.

4

Duplicate Resolution

Checks user's existing cart array. If product exists, performs quantity += incomingQuantity. Otherwise, pushes new item.

5

Dynamic Population & Response

Fetches live prices for all items in the newly updated cart, calculates totalAmount, and returns the populated payload to the client.

Order Checkout Flow

1

Client Checkout Initiation

App sends POST /api/orders with shippingAddress.

2

Cart Integrity Check

Backend fetches user. If user.cart.length === 0, throws a 400 Error to prevent empty "ghost" orders.

3

Price Snapshotting (Crucial)

Loops through user's cart and fetches the current live price from the Product table. Saves this exact number into the order items array. This permanently protects the user's receipt even if admin changes prices later.

4

Order Persistence

Creates new document in Order collection with status set to "pending" and calculated totalAmount.

5

Cart Purge

Executes user.cart = [] and saves to MongoDB. The client's cart is now officially emptied and the cycle is complete.
