API Reference

Complete documentation for the E-Commerce mobile application backend.

Base URL: https://saleh-ecommerce.up.railway.app

Authentication

Manage user registration and access tokens. Pass the returned JWT token as a Bearer token in the `Authorization` header for protected routes.

POST /api/auth/register

Registers a new user and returns a JWT token. Automatically initializes an empty cart for the user.

Request Body

FieldTypeDescription
name *stringFull name of the user
email *stringValid email address
password *stringMin 6 characters
cURL Request
curl -X POST https://saleh-ecommerce.up.railway.app/api/auth/register \
  -H "Content-Type: application/json" \
  -d '{
    "name": "Saleh Admin",
    "email": "saleh@example.com",
    "password": "password123"
  }'
201 Created Response
{
  "success": true,
  "data": {
    "user": { "name": "Saleh Admin", "email": "saleh@example.com", "id": "60d5ecb..." },
    "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9..."
  },
  "message": "User registered successfully"
}
POST /api/auth/login

Authenticates an existing user and returns a JWT token.

Request Body

FieldTypeDescription
email *stringRegistered email
password *stringUser's password
cURL Request
curl -X POST https://saleh-ecommerce.up.railway.app/api/auth/login \
  -H "Content-Type: application/json" \
  -d '{
    "email": "saleh@example.com",
    "password": "password123"
  }'

Products

The exact schema matches the `ProductResponseModel` used in the Flutter application.

GET /api/products

Retrieve a paginated list of all products.

Query Parameters

ParameterTypeDescription
pageintegerDefault: 1
limitintegerDefault: 20
categorystringFilter by exact category name
searchstringFuzzy search on product title
cURL Request
curl -X GET "https://saleh-ecommerce.up.railway.app/api/products?page=1&limit=10"
200 OK Response
{
  "success": true,
  "data": [
    {
      "id": 1,
      "title": "Fjallraven Backpack",
      "price": 109.95,
      "description": "Your perfect pack...",
      "category": "men's clothing",
      "image": "https://...",
      "rating": { "rate": 3.9, "count": 120 }
    }
  ],
  "message": "Products fetched successfully"
}
POST /api/products

Create a new product.

Request Body

FieldTypeDescription
id *intUnique integer ID
title *stringProduct name
price *floatItem price
description *stringItem description
category *stringCategory name
image *stringValid Image URL
rating.rate *floatReview score
rating.count *intTotal reviews
cURL Request
curl -X POST https://saleh-ecommerce.up.railway.app/api/products \
  -H "Content-Type: application/json" \
  -d '{
    "id": 1,
    "title": "New Jacket",
    "price": 55.99,
    "description": "Warm winter jacket.",
    "category": "men\'s clothing",
    "image": "https://example.com/img.png",
    "rating": { "rate": 4.5, "count": 10 }
  }'

Categories

GET /api/categories

Get a distinct list of all category strings extracted from the current products.

200 OK Response

    "jewelery",
    "men's clothing",
    "women's clothing"
  ],
  "message": "Categories fetched successfully"

Cart

All Cart endpoints require the Authorization: Bearer <token> header.

GET /api/cart

Retrieve the currently authenticated user's cart, populated with product details and totals.

Headers

HeaderValue
Authorization *Bearer {token}
cURL Request
curl -X GET https://saleh-ecommerce.up.railway.app/api/cart \
  -H "Authorization: Bearer YOUR_TOKEN_HERE"
200 OK Response
{
  "success": true,
  "data": {
    "items": [
      {
        "product": { "id": 1, "title": "Jacket", "price": 55.99, ... },
        "quantity": 2,
        "subtotal": 111.98
      }
    ],
    "total": 111.98
  },
  "message": "Cart fetched successfully"
}
POST /api/cart/add

Add a product to the user's cart. If the product already exists, the quantity will be increased.

Request Body

FieldTypeDescription
productId *intThe integer ID of the product
quantityintAmount to add (Default: 1)
cURL Request
curl -X POST https://saleh-ecommerce.up.railway.app/api/cart/add \
  -H "Authorization: Bearer YOUR_TOKEN_HERE" \
  -H "Content-Type: application/json" \
  -d '{
    "productId": 1,
    "quantity": 1
  }'
